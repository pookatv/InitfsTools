[description]
Graphics settings autodetection and quality presets - detects hardware capabilities and sets appropriate graphics quality levels for various visual features.

[version]
2011 (Battlefield 3 Alpha)
[notes]
Basic hardware detection with numeric quality values.
Simple Quality Metrics:
- Numeric values (0.0-3.0) instead of named presets
- Direct if-else chains based on GPU memory thresholds
Hardware Detection:
- cpuCount, cpuQuality + hardwareCpuBias
- gpuQuality + hardwareGpuBias
- availableGpuMemory for texture quality
Settings Covered:
- TextureQuality, MeshQuality, ShaderQuality
- AudioQuality, PhysicsQuality, AnimationQuality
Audio Threading:
- Audio.AudioCoreThread based on CPU core count
- Thread 0 for <4 cores, Thread 2 for 4-5 cores, Thread (n-1) for 6+ cores
GPU Memory Thresholds:
- ≤512MB: TextureQuality 0.0 (Low)
- ≤768MB: TextureQuality 1.0 (Medium)
- ≤1024MB: TextureQuality 2.0 (High)
- >1024MB: TextureQuality 3.0 (Ultra)
Limitations:
- No preset system
- Console-specific checks (Xenon/PS3)
- Limited configurability
[content]

	core.parseKeyValueCfgString(settings, _G) end

-- audio thread
if cpuCount < 4 then
	applySettings("Audio.AudioCoreThread 0")
else
	if (cpuCount < 6) then
		applySettings("Audio.AudioCoreThread 2")
	else
		applySettings("Audio.AudioCoreThread "..(cpuCount-1))
	end
end

-- audio
if cpuQuality < 4 then
    settings['AudioQuality'] = 0.0
else
    settings['AudioQuality'] = 1.0
end

-- physics
if cpuQuality < 4 then
    settings['PhysicsQuality'] = 0.0
else
    settings['PhysicsQuality'] = 1.0
end

-- animations
if cpuQuality < 4 then
    settings['AnimationQuality'] = 0.0
else
    settings['AnimationQuality'] = 1.0
end

-- mesh
if gpuQuality < 2 then
    settings['MeshQuality'] = 0.5
elseif gpuQuality < 2.5 then
    settings['MeshQuality'] = 1.0
elseif gpuQuality < 3 then
    settings['MeshQuality'] = 1.2
else
    settings['MeshQuality'] = 1.6
end

-- textures
if adapterName == "Xenon" or adapterName == "PS3" then
    settings['TextureQuality'] = 1.0
elseif availableGpuMemory <= 512*1024*1024 then
    settings['TextureQuality'] = 0.0
elseif availableGpuMemory <= 768*1024*1024 then
    settings['TextureQuality'] = 1.0
elseif availableGpuMemory <= 1024*1024*1024 then
	settings['TextureQuality'] = 2.0
else
    settings['TextureQuality'] = 3.0
end

-- shader
if hardwareProfile == 0 then
	settings['ShaderQuality'] = gpuQuality
else
	settings['ShaderQuality'] = hardwareProfile
end

[notes]
ARCHITECTURAL FOUNDATION - Established pattern used for next 14 years.
Critical Innovations:
1. makeConstant() - Type-safe enum system
2. defaultSettings.Render - Quality preset table structure
3. Low/Medium/High/Ultra nomenclature
4. Override system via overrideAutodetect.lua
Quality Enums Introduced:
- Quality: Autodetect(-1)/Low(0)/Medium(1)/High(2)/Ultra(3)/Custom(4)
- AnisotropicFilter: X1/X2/X4/X8/X16
- AntiAliasingDeferred: Off/MSAA2X/MSAA4X
- AntiAliasingPost: Off/Low/Medium/High
- AmbientOcclusion: Off/SSAO/HBAO
- MotionBlur: Off/On
Preset System Structure:
defaultSettings.Render[Quality.Low] = {
    TextureQuality, ShadowQuality, EffectsQuality,
    MeshQuality, TerrainQuality, UndergrowthQuality,
    AntiAliasingDeferred, AntiAliasingPost,
    AnisotropicFilter, AmbientOcclusion, MotionBlurEnabled
}
Settings Application:
1. Load preset for detected baseQualityLevel
2. Apply device-specific overrides
3. Allow user overrides via overrideAutodetect.lua
4. Save final values to Quality.Autodetect
Override System:
- Loads /user/profile/settings/overrideAutodetect.lua
- Uses metatable with __newindex to validate settings
- Allows constants (Quality.High) in override file
- Protected execution with pcall()
Impact:
Every subsequent version builds on this architecture. Without this redesign, system would remain a jumbled mess of if-statements.

function makeConstant(t)
	return setmetatable({}, 
	{
		__index = function (_, n)
			if t[n] == nil then
				log:error("Invalid constant member '" .. tostring(n) .. "'")

		__newindex = function (t, n, v)

		end
	})
end

RAY TRACING ERA BEGINS - Fundamental shift from rasterization to hybrid rendering.

1. RaytraceReflectionsQuality - First ray traced feature in Frostbite

- Clamps hardwareProfile to prevent overflow:
  if (hardwareProfile > (Quality.Ultra + 1)) then
      hardwareProfile = (Quality.Ultra + 1)
  end
Anti-Aliasing Simplification:
- AntiAliasingPost: Only Low(0) and High(1)
- Removed Medium tier for clarity
- MSAA4X only on Ultra/Cinematic
Ray Tracing Quality Levels:
- Low: RT reflections at low sample count
- Medium: Medium sample count
- High: High sample count
- Ultra: Maximum samples with denoising
Shadow Filtering:
- PCF (Percentage Closer Filtering): Fast, soft shadows
- PCSS (Percentage Closer Soft Shadows): Realistic, contact-hardening
GPU Memory Management:
- Still uses 1.5GB/2.5GB/4GB+ thresholds
- ClampGpuMemory setting added
System Memory Checks:
- Commented-out logic to reduce quality on low RAM
- Planned for future but not active
Base Quality Override:
- baseQualityLevel can override autodetect
- Allows command-line quality forcing
Impact:
Opened door for photorealistic lighting. Every game after considers ray tracing. Single biggest visual technology shift in document. DXR requirement changed PC hardware landscape.

	if lookupTable == nil then

	end
	local result = Quality.Ultra
	for _, tableValue in ipairs(lookupTable[desktopResolution]) do

			break end
		result = result - 1
	end
	return result
end
