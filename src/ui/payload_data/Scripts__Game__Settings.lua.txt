[description]
Core settings initialization - defines fundamental game configuration tables loaded before everything else.

[version]
2011-2016 (Battlefield 3 (Alpha), NFS, Battlefront 1, Garden Warfare 1/2, Mirror's Edge)
[notes]
Original comprehensive settings structure.

Features:
- Game.LogFileEnable = true
- GameMode.disablePreRound = true
- Client, Server, ShaderSystem tables
- NetworkSettings with full configuration
- Core table (empty, placeholder for future expansion)

This was the most feature-rich early version, establishing patterns used throughout Frostbite's history.

[version]
2025+ (Battlefield 6, NFL 26, Skate)
[notes]
REVOLUTIONARY ARCHITECTURAL CHANGE: String-based configuration.

This represents the most significant Settings.lua change in Frostbite history - a complete paradigm shift from Lua tables to declarative string-based configuration.

Major Changes:
- core = require 'Frost.Core' - Module import
- applySettings() function wrapper
- Configuration as multi-line string: applySettings [=[ ... ]=]
- Single setting: Core.AllowSettingsParseErrors false
- parseKeyValueCfgString() with location parameter: 'Settings.lua'

Architectural Philosophy:
- Declarative over imperative
- Text-based over table-based
- Easier to parse, validate, and generate
- Better separation between code and data
- Consistent format with other .cfg files

Why This Changed Everything:
1. Easier tooling - can generate configs without Lua knowledge
2. Better validation - parseKeyValueCfgString enforces format
3. Version control friendly - clear text diffs
4. Cross-language compatible - any language can generate these strings
5. Unified format - Settings.lua now matches Game.cfg, local.cfg, etc.

This is the future of Frostbite configuration - a complete rewrite that fundamentally changes how game settings are defined.
[content]
--
-- Define basic settings tables
--

core = require "Frost.Core"

local function applySettings(settings)
	core.parseKeyValueCfgString("Settings.lua", settings, _G)
end

applySettings [=[
Core.AllowSettingsParseErrors false
]=]
[/version]
